# AI Agent Demo Project

このリポジトリは、**GitHub Issueを立てるだけでAIが実装してPull Requestを作成してくれる**仕組み(DevOps AI Agent)のお試し用リポジトリです。

## これは何?

1. このリポジトリに Issue を作成する
2. AIが内容を読んで、実装(コード修正)を行う
3. AIが自動でブランチを切り、Pull Requestを作成する

裏側では [ai-agent-demo-genarator](https://github.com/TakuKobayashi/ai-agent-demo-genarator) というツールが、Google Cloud上で動いています。**あなた自身がGoogle Cloudのアカウントを用意したり、何かのトークンを発行したりする必要は一切ありません。** 下記の「GitHub Appをインストールする」を1回行うだけで使えるようになります。

---

## セットアップ手順 (これだけでOK)

### ① GitHub App をインストールする

1. 以下のインストールページを開く (URLは運営者から共有されたものを使用してください)
   ```
   https://github.com/apps/<App名>/installations/new
   ```
2. インストール先のアカウント(自分のユーザーアカウント、または所属組織)を選ぶ
3. 「Repository access」で、AIに使わせたいリポジトリを選ぶ
   - **Only select repositories** を選び、このリポジトリ (`ai-agent-demo-genarator-demo-project`) にチェックを入れるのがおすすめです (全リポジトリ許可は非推奨)
4. 「Install」をクリックして完了

これだけで準備は完了です。GCPのアカウント作成、APIキーの発行、Personal Access Token (PAT) の発行、Webhookの手動設定などは**一切不要**です。

> **補足**: なぜこれだけでよいのか?
> このシステムは GitHub App という仕組みを使っています。GitHub Appを運営者(1人)が事前に作成し、Google Cloud上のサーバーに繋げてあるため、あなたがインストールするだけで、あなたのリポジトリからのIssueイベントが自動的にそのサーバーに届くようになります。コード変更の反映(git push・PR作成)に必要な権限も、GitHub Appの「そのリポジトリだけに使える・有効期限が短いトークン」がその都度自動発行される仕組みになっており、あなたが何か秘密の情報を渡す必要はありません。

### ② (必要な場合のみ) 運営者にリポジトリを伝える

運営者側でセキュリティのため利用可能なリポジトリを限定している場合があります。①でインストールしても後述の「動作確認」でPRが作成されない場合は、運営者にこのリポジトリ名 (`<あなたのアカウント名>/ai-agent-demo-genarator-demo-project` のようなフルネーム) を伝えて、許可リストに追加してもらってください。

## 動作確認

1. このリポジトリの **Issues** タブを開く
2. 「New issue」で、実装してほしい内容をタイトル・本文に書いて作成する
   - 例: タイトル「READMEにライセンスバッジを追加」、本文に具体的な要望を書く
   - シンプルで小さい要望ほど成功しやすいです (大きすぎる要望はAIが対応しきれない場合があります)
3. 数分待つ (Geminiでのコード生成とgit操作に少し時間がかかります)
4. **Pull requests** タブを確認する。AIが作成したPRが増えていれば成功です

## うまく動かないとき

| 症状 | 確認すること |
|---|---|
| Issueを作ってもPRが全く作られない | ① GitHub Appがこのリポジトリにインストールされているか (Settings → Integrations → GitHub Apps で確認できます) |
| 同上 | ② 運営者側で許可リポジトリが制限されていないか。運営者に確認・依頼してください |
| PRは作られたが中身が期待と違う | Issueの本文をもっと具体的に書いてみてください (対象ファイル名、期待する挙動など) |
| 何回試してもエラーになる | 運営者に、対象のIssue番号と時刻を伝えて、サーバー側のログを確認してもらってください |

## このAppが持っている権限について

GitHub Appは以下の権限だけを持つよう設定されています(それ以外の操作は一切できません):

- **Contents**: リポジトリの内容の読み書き (ブランチ作成・コミット・push用)
- **Pull requests**: Pull Requestの作成・読み取り
- **Issues**: Issueの読み取り (内容を把握するため)

リポジトリの削除・Settings変更・他のリポジトリへのアクセスなどはできません。

## インストールを取りやめたい場合

GitHubの Settings → Applications → Installed GitHub Apps から、いつでもこのAppのインストールを解除できます。解除すると、以降このリポジトリのIssueに反応しなくなります。
